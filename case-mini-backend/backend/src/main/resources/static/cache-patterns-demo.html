🎯 Cache Patterns Demo

Test Read-through, Write-through/Invalidation và Anti-Stampede patterns

🔍 Read-through Cache

PATTERN FLOW:
GET case/{id} → Cache hit? → Return cached
               │
               Cache miss → Query DB → Cache result → Return

✏️ Write-through / Invalidation

PATTERN FLOW:
UPDATE case/{id} → Save to DB → Invalidate cache
                                         │
Next GET → Cache miss → Fresh data from DB

🛡️ Anti-Stampede

STAMPEDE SCENARIO:
5 concurrent requests → 1 key → Cache miss
Without lock: 5 DB queries 😱
With lock: 1 DB query + 4 wait 😎
